Requirement Summary

Uninstallation Best Practices

1. Single Node: Used for maintenance or scaling down. The uninstaller removes the node from the cluster metadata before purging local binaries. Quorum must be maintained (n/2 + 1).

2. Cluster-Wide: Used for full environment migration or decommissioning. This triggers a global wipe of the configuration store and node records.

Active Directory Note
Removing VLauncher from a server does not automatically delete the AD Computer Object. Cleanup of the OU: Vlauncher objects should be verified manually.
